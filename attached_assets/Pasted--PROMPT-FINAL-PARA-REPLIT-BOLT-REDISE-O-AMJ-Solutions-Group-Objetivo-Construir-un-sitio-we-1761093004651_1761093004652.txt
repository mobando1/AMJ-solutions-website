# PROMPT FINAL PARA REPLIT/BOLT – REDISEÑO `AMJ Solutions Group`

## Objetivo

Construir un sitio web corporativo **moderno, elegante y creíble** para **AMJ Solutions Group** (consultora de comunicación corporativa, manejo de crisis y reputación, con sede en Boca Ratón, Florida, fundada por **Ana Nelson**). El sitio debe transmitir **autoridad**, **claridad** y **confianza**, optimizado para **mobile-first** y SEO básico.

> **Fuente de contenidos**: Usa **todo el copy** y datos (email, teléfono, dirección, servicios, biografía, etc.) **desde el archivo que subiré a este proyecto** (`/content/amj_copy.md`) o, si se provee como texto en el prompt de ejecución, **parsea** ese texto y distribúyelo en las secciones correspondientes. **No inventes datos** si faltan: utiliza placeholders claramente marcados.

> **Activos**: Subiré el **logo** en SVG/PNG a `/public/brand/` y los **logos de clientes** (SVG o PNG transparente) a `/public/clients/`.

---

## Stack técnico (preferido)

* **Next.js 14** (App Router) + **TypeScript**
* **Tailwind CSS** (con `@tailwindcss/typography` y `@tailwindcss/forms`)
* **Shadcn/ui** componentes básicos (botones, cards, dialog) – opcional
* **next-seo** para metatags por página
* **next/image** para optimización de imágenes
* **ESLint + Prettier**
* **Vercel-ready** (config de `headers`, `redirects` si aplica)

Si no es posible Next.js, generar una variante **Vite + React + TS** con estructura equivalente y comentarios de dónde ajustar.

## Estilo visual

* **Tono**: profesional, elegante, ejecutivo, minimalista.
* **Paleta sugerida** (editable):

  * `--brand-bg`: `#FFFFFF`
  * `--brand-text`: `#1F2937` (gris muy oscuro)
  * `--brand-accent`: `#1C3D4F` (azul petróleo) **o** `#0F4C3A` (verde sobrio) – usar **una** de acento
  * `--brand-muted`: `#6B7280`
  * `--brand-border`: `#E5E7EB`
* **Tipografía**: `Poppins` o `Lato` (Google Fonts). Títulos semi-bold; cuerpo regular; interlineado cómodo (1.6–1.8).
* **Fotografía**: fondos neutros/corporativos; desaturados sutiles; recortes limpios.
* **Iconografía**: minimal, trazo fino.
* **Micro-interacciones**: fades/scale muy sutiles (duración 150–200ms). No obstaculizar la lectura.

## Componentes clave

### `CalendlyButtonModal`

* Botón primario que abre **Dialog** con **embed de Calendly** (`CALENDLY_URL`).
* Fallback: si no hay URL, el botón muestra tooltip "Calendly URL pending".

### `LogoWall`

* Lee todas las imágenes de `/public/clients/*` y rinde un **grid responsive** (2–4–6 columnas según breakpoint).
* Cada logo con `next/image`, `alt` descriptivo (tomar del nombre de archivo a texto legible).
* **Grayscale** al reposo y **fade-in color** al hover (opcional).

### `ServiceCard`

* Título, breve descripción (2–3 líneas), bullets de outcomes, CTA a Calendly.

### `ContactForm`

* Validación con `zod` + `react-hook-form`.
* Estados de envío; **no** enviar a ningún servicio externo (placeholder).
* Accesible: labels, `aria-live` para mensajes de estado.

## SEO básico y metadatos

* **next-seo** por página (title ≤ 60, description 150–160).
* **Open Graph** y **Twitter Card** (`summary_large_image`) con `og:image` desde `/public/og/og-default.jpg`.
* **Favicons**: 32/48/64 y `site.webmanifest`.
* **Schema.org**:

  * `Organization` (AMJ Solutions Group)
  * `Person` (Ana Nelson, `worksFor` AMJ)
  * `Service` por cada servicio principal
  * (Opc.) `FAQPage` si agregamos sección de preguntas
* **URLs limpias** y **slugs** kebab-case para anchors de servicios.

## Accesibilidad & performance

* Contraste mínimo **WCAG AA** (usa utilidad Tailwind para contrastes).
* Estados de `:focus` visibles y navegación por teclado.
* `alt` en todas las imágenes (logos incluidos).
* `next/image` con tamaños responsivos (`sizes`) y `loading="lazy"` por defecto.
* Fuentes cargadas con `display: swap`; preconnect a Google Fonts.

---

## Responsive (desktop + mobile)

* **Mobile-first**:

  * Hero con tipografía que no rompa (clamp).
  * Grids colapsan a 1 columna; paddings generosos (`px-4`/`px-6`).
  * Botón Calendly **sticky** opcional en mobile (bottom bar) – pointer large.
* **Desktop**:

  * Máximo ancho `max-w-screen-xl`.
  * Secciones con `py-20` aprox.; jerarquía clara de titulares.
